### 1.3. Жёсткие правила по цветам

- **ЗАПРЕЩЕНО** использовать произвольные цвета Tailwind: `bg-blue-500`, `text-gray-600`, `border-slate-200` и т.п.
- **ОБЯЗАТЕЛЬНО** использовать только CSS-переменные: `bg-primary`, `text-muted-foreground`, `border-border` и т.п.
- **ЗАПРЕЩЕНО** использовать inline-стили для цветов: `style={{ color: '#2563EB' }}`
- Статусные цвета применять через утилитарные классы (определены в `index.css`):
  - Текст: `.text-status-active`, `.text-status-warning`, `.text-status-danger`, `.text-status-info`, `.text-status-neutral`
  - Фон: `.bg-status-active`, `.bg-status-warning`, `.bg-status-danger`, `.bg-status-info`
  - Фон 10% opacity: `.bg-status-active/10`, `.bg-status-warning/10`, `.bg-status-danger/10`, `.bg-status-info/10`
  - Рамка 30% opacity: `.border-status-active/30`, `.border-status-warning/30`, `.border-status-danger/30`, `.border-status-info/30`
  - **ЗАПРЕЩЕНО** использовать inline-HSL: `text-[hsl(var(--status-active))]` — только утилитарные классы

## 2. ТИПОГРАФИКА

### 2.1. Шрифт

- Основной: `Inter` (подключить через Google Fonts или локально)
- Моноширинный (для IP-адресов, fingerprint, кода): `JetBrains Mono`
- Запасной: `system-ui, -apple-system, sans-serif`

### 2.2. Размеры текста

| Элемент | Размер | Вес | Класс Tailwind |
|---------|--------|-----|----------------|
| Заголовок страницы (h1) | 24px | 700 (bold) | `text-2xl font-bold` |
| Заголовок секции (h2) | 18px | 600 (semibold) | `text-lg font-semibold` |
| Заголовок карточки (h3) | 14px | 600 (semibold) | `text-sm font-semibold` |
| Основной текст | 14px | 400 (normal) | `text-sm` |
| Подпись, хинт | 12px | 400 (normal) | `text-xs text-muted-foreground` |
| Крупное число (KPI) | 24px | 700 (bold) | `text-2xl font-bold` |
| Число в ячейке таблицы | 14px | 500 (medium) | `text-sm font-medium tabular-nums` |
| Бейдж | 12px | 500 (medium) | `text-xs font-medium` |

### 2.3. Правила

- Числа в таблицах всегда с `tabular-nums` для выравнивания по столбцам
- Даты: формат `дд.мм.гггг` или `дд.мм.гг`, время `чч:мм`
- IP-адреса и fingerprint: моноширинный шрифт `font-mono`
- Не использовать `text-lg` и выше для обычного текста
- Не использовать `font-bold` для обычного текста — только для заголовков и KPI

## 3. КОМПОНОВКА И ПЛОТНОСТЬ

### 3.1. Главный принцип

**Это рабочий инструмент для администраторов и операторов, а НЕ маркетинговый сайт.**
Каждый пиксель должен нести информацию. Пустое пространство допускается только для визуального разделения логических блоков, но НЕ для «воздуха» и «дыхания» макета.

### 3.2. Сетка страницы

```
+---------------------------------------------------+
| [Sidebar 240px]  |  [Content area, fluid]          |
|                  |                                  |
|  Logo            |  +- Page header ---------------+ |
|  -----           |  | H1 + описание   [Кнопки]    | |
|  Nav item        |  +-----------------------------+  |
|  Nav item        |                                  |
|  Nav item        |  [Контент страницы]              |
|  ...             |                                  |
|                  |                                  |
|  -----           |                                  |
|  User menu       |                                  |
+---------------------------------------------------+
```

- **Sidebar**: фиксированная ширина 240px, сворачивается до 64px (только иконки)
- **Content area**: `max-w-full`, padding `px-6 py-4`
- **Не использовать** `max-w-7xl`, `max-w-screen-xl`, `container` — контент должен занимать всю доступную ширину

### 3.3. Отступы (spacing)

| Контекст | Значение | Класс |
|----------|----------|-------|
| Padding страницы | 24px горизонт., 16px вертик. | `px-6 py-4` |
| Между секциями (блоками) | 16px | `gap-4` или `space-y-4` |
| Padding внутри карточки | 16px | `p-4` |
| Padding компактной карточки | 12px | `p-3` |
| Между элементами в карточке | 8px | `gap-2` или `space-y-2` |
| Между строками таблицы | 0 (только border) | — |
| Padding ячейки таблицы | 8px вертик., 12px горизонт. | `px-3 py-2` |
| Между кнопками в группе | 8px | `gap-2` |
| Padding кнопки | 8px вертик., 16px горизонт. | `px-4 py-2` |

**ЗАПРЕЩЕНО:**
- `p-8`, `p-10`, `p-12` и выше внутри карточек и контейнеров
- `gap-6`, `gap-8` и выше между карточками
- `py-6`, `py-8` между секциями
- `mb-8`, `mb-10` после заголовков
- Пустые контейнеры и разделители ради «воздуха»

### 3.4. Карточки статистики (KPI-карточки)

**ПРАВИЛЬНО** — компактные карточки:
```
+--------------+ +--------------+ +--------------+ +--------------+
| * 3          | | ! 0          | | ^ 0          | | = 3          |
| Активных     | | С ошибками   | | Нагрузка     | | Всего        |
+--------------+ +--------------+ +--------------+ +--------------+
```

- Высота карточки: auto (контент определяет)
- Padding: `p-3` (12px)
- Число: `text-2xl font-bold` (24px)
- Подпись: `text-xs text-muted-foreground` (12px)
- Иконка статуса: цветная точка 8px или маленькая иконка 16px рядом с числом
- Сетка: `grid grid-cols-4 gap-3` (или `grid-cols-2 sm:grid-cols-4`)
- Нет тени, только `border border-border`

**НЕПРАВИЛЬНО (антипаттерн):**
```
+------------------------------+
|                              |
|        [icon]                |
|                              |
|        1                     |
|                              |
|    Всего серверов            |
|                              |
+------------------------------+
```

Такие карточки ЗАПРЕЩЕНЫ — гигантская иконка, огромные отступы, одна цифра на огромном пространстве.

### 3.5. Таблицы

Таблица — основной элемент интерфейса PAM. Правила:

- **Padding ячеек**: `px-3 py-2` (12px / 8px)
- **Шрифт**: `text-sm` (14px)
- **Заголовки столбцов**: `text-xs font-medium text-muted-foreground uppercase tracking-wide`
- **Hover строки**: `hover:bg-accent` (лёгкая подсветка)
- **Zebra-stripe**: НЕ использовать. Разделение — только горизонтальные линии `border-b border-border`
- **Числовые столбцы**: выравнивание `text-right tabular-nums`
- **Текстовые столбцы**: выравнивание `text-left`
- **Статус в таблице**: маленький бейдж или цветная точка + текст, НЕ большой цветной блок
- **Действия**: иконки без текста в последнем столбце, или меню-dropdown (три точки)
- **Пагинация**: внизу таблицы, компактная, показывать «Показано 1-25 из 340»
- **Пустое состояние**: одна строка текста по центру: «Нет данных» — без гигантских иллюстраций

**Минимальное количество столбцов**: стремиться показать максимум данных. Если данные есть — показывать их в таблице, а не прятать на отдельные страницы.

### 3.6. Формы

- **Лейблы**: `text-sm font-medium`, над полем ввода, отступ `mb-1.5`
- **Поле ввода**: высота `h-9` (36px), padding `px-3`, `text-sm`
- **Расстояние между полями**: `gap-4` (16px)
- **Горизонтальная раскладка**: для коротких полей (порт, тип) использовать `grid grid-cols-2 gap-4` или `grid-cols-3`
- **Кнопки формы**: в правом нижнем углу, `gap-2` между ними
- **Обязательные поля**: красная звёздочка `*` после лейбла
- **Ошибки валидации**: `text-xs text-destructive mt-1` под полем

### 3.7. Модальные окна (Dialogs)

- Ширина: `sm:max-w-md` для простых форм, `sm:max-w-lg` для сложных
- Padding: `p-6`
- Заголовок: `text-lg font-semibold`
- Кнопки: внизу справа, основная `variant="default"`, вторичная `variant="outline"`
- Не использовать модалки для отображения больших таблиц — для этого нужна отдельная страница

## 4. КНОПКИ И ИНТЕРАКТИВНЫЕ ЭЛЕМЕНТЫ

### 4.1. Варианты кнопок

| Вариант | Использование | Класс shadcn |
|---------|--------------|--------------|
| Primary (filled) | Основное действие: «Добавить», «Сохранить» | `variant="default"` |
| Secondary (outlined) | Второстепенное: «Отмена», «Фильтр», «Экспорт» | `variant="outline"` |
| Ghost | Навигация, иконки в тулбаре | `variant="ghost"` |
| Destructive | Удаление сервера, блокировка пользователя | `variant="destructive"` |
| Link | Текстовые ссылки внутри контента | `variant="link"` |

### 4.2. Размеры кнопок

| Размер | Где использовать | Класс |
|--------|-----------------|-------|
| Default (h-9) | Большинство случаев | `size="default"` |
| Small (h-8) | В таблицах, тулбарах, компактных формах | `size="sm"` |
| Icon (h-9 w-9) | Кнопки-иконки | `size="icon"` |

- **ЗАПРЕЩЕНО** использовать `size="lg"` — это для лендингов, не для рабочих интерфейсов
- На каждой странице должна быть **не более 1-2 primary-кнопок**
- Primary-кнопка — всегда в правом верхнем углу или правом нижнем углу формы
- **ОБЯЗАТЕЛЬНО**: `aria-label` на каждой icon-only кнопке (`size="icon"`) для accessibility

### 4.3. Расположение кнопок

- **Шапка страницы**: primary-кнопка справа (напр. «+ Добавить сервер»)
- **Форма**: кнопки внизу справа, порядок: [Отмена] [Сохранить]
- **Таблица**: кнопки действий — иконки `size="icon" variant="ghost"` в последнем столбце
- **Тулбар (фильтры)**: `variant="outline" size="sm"`
- **Между кнопками**: `gap-2` (8px)

### 4.4. Иконки

- Библиотека: **Lucide React** (уже входит в shadcn/ui)
- Размер иконок: `16px` (`w-4 h-4`) — стандарт, `20px` (`w-5 h-5`) — в навигации
- Иконки рядом с текстом: `gap-2` между иконкой и текстом
- Не использовать иконки размером `24px` и выше в карточках статистики
- Не использовать эмодзи вместо иконок

## 7. СТАТУСЫ И БЕЙДЖИ

### 7.1. Цвета статусов

| Статус | Цвет | Применение в PAM |
|--------|------|-----------------|
| Активен | Зелёный (`status-active`) | Сервер работает, БД активна, пользователь активен |
| Предупреждение | Янтарный (`status-warning`) | Высокая нагрузка, мало места на диске, статичные подключения |
| Недоступен / Ошибка | Красный (`status-danger`) | Сервер недоступен, ошибка подключения, БД удалена |
| Информация | Синий (`status-info`) | Новый сервер, ожидание данных |
| Неизвестен | Серый (`status-neutral`) | Статус неизвестен, нет данных |

### 7.2. Формат бейджей

- Размер: `text-xs font-medium px-2 py-0.5 rounded-full`
- Стиль: **мягкий фон** + тёмный текст (НЕ яркий залитый бейдж)
  - Пример: `bg-green-100 text-green-800 dark:bg-green-900/30 dark:text-green-400`
- В таблицах можно использовать точку-индикатор вместо бейджа:
  - `<span className="w-2 h-2 rounded-full bg-status-active inline-block" />`

## 8. ГРАФИКИ И ВИЗУАЛИЗАЦИЯ

### 8.1. Библиотека

Использовать **Chart.js** с адаптером `chartjs-adapter-date-fns` и русской локалью.

### 8.2. Правила

- Цвета графиков: только `--chart-1` ... `--chart-5`
- Высота графика: `h-48` (192px) для обычных, `h-64` (256px) для основных
- Фон: прозрачный (карточка даёт фон)
- Сетка (grid lines): тонкая, адаптивная к теме (светлая на тёмном, тёмная на светлом)
- Подписи осей: `text-xs text-muted-foreground`
- Tooltip: тёмный фон, белый текст, скруглённые углы, полная дата+время
- Легенда: только если >2 серий данных, внизу графика
- Градиент заливки: от цвета линии (25% opacity) к прозрачному
- **ЗАПРЕЩЕНО**: 3D-эффекты, градиенты на столбцах, тени, анимации загрузки дольше 300ms

### 8.3. Форматы осей времени

|--------|------|-----------|

| до 30 дней | `day` | `d MMM` |
| до 180 дней | `week` | `d MMM` |

### 10.1. Загрузка (Loading)

- Skeleton: использовать shadcn `<Skeleton />` компонент
- Показывать skeleton того же размера, что и контент
- НЕ показывать полноэкранные спиннеры
- Для таблиц: 5 строк skeleton

### 10.2. Пустые состояния

- Одна иконка `w-8 h-8 text-muted-foreground`
- Заголовок: `text-sm font-medium`
- Подзаголовок: `text-xs text-muted-foreground`
- Кнопка действия: `variant="outline" size="sm"`
- Всё по центру в пустой области
- **ЗАПРЕЩЕНО**: гигантские иллюстрации, большие анимации, текст больше 2 строк

## 12. АНТИПАТТЕРНЫ — ЧТО ЗАПРЕЩЕНО ДЕЛАТЬ

Этот раздел — контрольный список. Если Claude Code генерирует что-то из списка ниже, это нужно исправить.

### Категорически запрещено:

1. **Гигантские карточки с одной цифрой** — если в карточке одно число и подпись, padding не должен быть больше `p-3`
2. **Иконки > 20px в карточках статистики** — максимум `w-5 h-5` (20px)
3. **Пустое пространство больше 24px между блоками** — максимум `gap-4` (16px)
4. **`max-w-7xl` или `container` для контента** — контент занимает всю ширину
5. **Произвольные цвета** (`bg-blue-500`, `text-gray-600`) — только CSS-переменные
6. **Большие кнопки** (`size="lg"`) — только `default` и `sm`
7. **Более 2 primary-кнопок на странице**
8. **Тень на карточках** (`shadow-lg`, `shadow-md`) — максимум `shadow-sm` или без тени
9. **Скруглённые углы больше** `rounded-lg` — не использовать `rounded-xl`, `rounded-2xl`
10. **Полноэкранные пустые состояния** с огромными картинками
11. **Vertical spacing > 16px** (`space-y-6`, `space-y-8`, `gap-6`, `gap-8`) между карточками
12. **Padding > 16px** (`p-6`, `p-8`) внутри карточек-виджетов
13. **Центрирование текста** в карточках статистики — текст всегда `text-left`
14. **Hero-секции, баннеры, welcome-блоки** — это рабочий интерфейс, не лендинг
15. **Эмодзи как иконки** — использовать только Lucide React

## 13. КОНТРОЛЬНЫЙ ЧЕКЛИСТ ДЛЯ КАЖДОЙ СТРАНИЦЫ

Перед завершением работы над страницей, Claude Code должен проверить:

- [ ] Все цвета используют CSS-переменные (нет произвольных)
- [ ] Padding карточек <= `p-4`
- [ ] Spacing между блоками <= `gap-4`
- [ ] Заголовок страницы — `text-2xl font-bold`, не больше
- [ ] Primary-кнопка не более 1-2 на странице
- [ ] Таблицы с `text-sm`, padding ячеек `px-3 py-2`
- [ ] Числа с `tabular-nums`
- [ ] Статусы показаны бейджами или точками, единообразно
- [ ] Контент использует полную ширину (нет `max-w-*` ограничений)
- [ ] Нет элементов > 24px spacing
- [ ] Страница вмещается без лишней прокрутки на экране 1920x1080
- [ ] Dark mode корректно отображается
- [ ] Loading state — skeleton, не спиннер
- [ ] Empty state — компактный, не полноэкранный

# PostgreSQL Activity Monitor

Этот проект предназначен для мониторинга количества подключений к базе данных. Это позволяет определить, какие базы данных активно используются, а какие давно не применялись.

## 📌 Функциональность
- Подключение к нескольким базам данных.
- Отслеживание количества активных подключений.
- Определение неиспользуемых баз данных.
- Авторизация пользователей через JSON-файл.

## 🛠️ Установка и настройка
### 1. Клонирование репозитория
```sh
git clone <репозиторий>
cd <папка_проекта>
```

### 2. Настройка конфигурации
В файле `servers.json` указываются базы данных, за которыми ведётся мониторинг:
```json
{
    "servers": [
        {
            "name": "s00-dbs06",
            "host": "10.110.125.61",
            "user": "postgres",
            "password": "",
            "port": 5432
        }
    ]
}
```

Файл `users.json` содержит зашифрованные пароли для авторизации:
```json
{
    "admin": "$2y$10$HZRcoL1CZm9TpxJ/8Xv3WOxhWIAm2Ik9QP75Me7ScubF1Hvy043RC"
}
```

### 3. Запуск проекта
Разместите файлы на веб-сервере и откройте `index.php` в браузере.

## 📂 Файловая структура
```
📂 project_root
├── config.php     # Конфигурационный файл
├── index.php      # Главная страница мониторинга
├── login.php      # Форма авторизации
├── logout.php     # Выход из системы
├── servers.json   # Список баз данных
└── users.json     # Список пользователей
```

## 🔐 Авторизация
Для входа в систему используется `login.php`. Пароли хранятся в `users.json` в зашифрованном виде.
